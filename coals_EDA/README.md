# 🧘‍♀️ AI 필라테스 코치 - 실시간 관절 추출 및 각도 분석 (coals_EDA)

본 폴더는 캡스톤 프로젝트의 핵심인 **'컴퓨터 비전(AI Vision) 기반 자세 평가 파이프라인'**의 연구 및 구현 결과를 담고 있습니다. 동영상을 입력받아 사용자의 관절을 실시간으로 추적하고, 수학적 각도 계산을 통해 자세 피드백을 제공합니다.

---

## 📊 사용 데이터 및 타겟 동작

- **데이터 출처:** AI Hub 개방 데이터 - **[216. 필라테스 동작 데이터]**
- **타겟 동작:** 매트(Mat) 기구 - **'척추 스트레칭 (Spine Stretch)'** / 난이도: 고급
- **데이터 활용:** 원천 데이터에 포함된 8방향 다각도 카메라(CAM_1 ~ CAM_8) 영상 중, 앞뒤 움직임(시상면) 분석에 가장 적합한 **측면 카메라(CAM_3, CAM_4)** 영상을 선별하여 테스트 및 검증에 활용하였습니다.

---

## 🚀 핵심 구현 기능

1. **실시간 관절 추적 (Pose Estimation):** 가볍고 빠른 `YOLOv8n-pose` 모델을 사용하여 영상 속 객체의 17개 관절(Keypoints) 좌표(X, Y)를 실시간으로 추출합니다.
2. **수학적 각도 계산 로직:** 골반(Hip) - 무릎(Knee) - 발목(Ankle)의 2D 좌표를 삼각함수(`arctan2`)를 이용해 계산하여 실제 관절의 굽힘 각도를 측정합니다.
3. **규칙 기반(Rule-based) 코칭 피드백:** - 무릎 각도가 160도 미만일 경우 ➜ 🔴 `WARNING: Straighten your knee!` (무릎 폄 경고 출력)
   - 무릎 각도가 170도 이상일 경우 ➜ 🔵 `GOOD POSTURE` (올바른 자세 칭찬 출력)

## 💡 핵심 인사이트 및 트러블슈팅 (Troubleshooting)

### 🚨 문제 발생: 2D 투영 오차 (Projection Error)

초기 테스트 시, 실제로는 무릎을 완전히 폈음에도 불구하고 AI가 각도를 '32도'로 아주 좁게 인식하는 치명적인 오류가 발생했습니다.
원인 분석 결과, 2D 카메라(웹캠, 스마트폰 등)는 깊이(Depth, Z축)를 인식하지 못하기 때문에, **후면이나 대각선에서 촬영된 영상의 경우 화면상에서 골반-무릎-발목의 Y좌표 차이가 거의 겹쳐 보이면서 발생하는 2D 카메라의 근본적인 한계**임을 밝혀냈습니다.

### 🎯 해결 방안: 동작별 최적의 카메라 시점(View) 매핑

척추 스트레칭(Spine Stretch)처럼 몸이 앞뒤(시상면, Sagittal Plane)로 움직이는 동작은 **반드시 측면(Side-view) 카메라 영상을 사용하여 분석**해야 합니다.
이를 위해 대용량 원천 데이터 중 후면 영상이 아닌 측면 카메라 파일만 자동으로 추출하여 파이프라인에 주입하도록 스크립트(`find_video.py`)를 개선하였고, 결과적으로 170도 이상의 정확한 무릎 각도 측정이 가능해졌습니다.

## 📁 주요 파일 설명

- `yolo_coach.py`: **[핵심 실행 파일]** 영상을 불러와 YOLOv8로 관절을 찾고, 각도를 계산하여 화면에 실시간 피드백을 렌더링하는 메인 스크립트입니다.
- `find_video.py`: 대용량 원천 데이터 폴더 구조를 탐색하여 특정 시점(측면 카메라 등)의 영상만 찾아 복사해 주는 유틸리티 스크립트입니다.
- `benchmark_models.py` / `run_eda.py` 등: 원천 데이터의 스키마를 확인하고 뼈대 데이터를 탐색적 데이터 분석(EDA)하기 위한 연구용 스크립트 모음입니다.
- `*_Report.png`: EDA 분석 결과를 시각화한 리포트 이미지입니다.
